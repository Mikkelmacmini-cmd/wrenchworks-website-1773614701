# Website Production SOP (Living Document)

_Last updated: 2026-03-10 (MDT)_
_Owner: AX_

## Purpose
Ship premium, modern, high-converting local-service websites with consistent quality and speed.

## Non-Negotiables
- Brand standard: **Premium / Bold / Trustworthy**
- No code-first visual guessing
- No launch without passing quality gates
- Sentinel runs once at the end (runtime-capped)

---

## Phase 1 — Strategy Lock
**Owners:** AX + Atlas  
**Output:** `strategy-brief.md`

### Checklist
- [ ] ICP and local market clearly defined
- [ ] Core offer + primary conversion action defined
- [ ] Trust proof inventory collected (reviews, warranties, certifications)
- [ ] SEO + AI-search intent map drafted
- [ ] Non-negotiable brand adjectives locked

**Gate:** No design work until this is approved.

---

## Phase 2 — Concept Sprint (3 Directions)
**Owners:** Muse + Lens  
**Output:** `concept-a.md`, `concept-b.md`, `concept-c.md`

### Checklist
- [ ] 3 visibly distinct concepts (not minor variants)
- [ ] Hero concept for each direction
- [ ] Color, typography, spacing, and component language
- [ ] Mobile-first layout examples
- [ ] Imagery/icon treatment notes

**Gate:** Direction selected before implementation.

---

## Phase 3 — Design Critic Gate
**Owner:** Vanguard  
**Output:** `vanguard-scorecard.md`

### Scoring Rubric (100)
- [ ] Premium perception (20)
- [ ] Bold visual identity (20)
- [ ] Trustworthiness signals (20)
- [ ] 2026 modernity/originality (20)
- [ ] Conversion clarity (20)

**Pass threshold:** 85+ and no P0s.

---

## Phase 4 — Content + Interaction + SEO Alignment
**Owners:** Quill + Pulse + Atlas

### Checklist
- [ ] Quill: headline hierarchy + CTA ladder finalized
- [ ] Pulse: interaction/motion spec finalized (accessible)
- [ ] Atlas: metadata/schema/internal linking finalized
- [ ] Messaging and visual direction are consistent

---

## Phase 5 — Build Implementation
**Owner:** Forge  
**Output:** implementation summary + build results

### Checklist
- [ ] Approved concept translated accurately
- [ ] Route/index hygiene preserved
- [ ] Accessibility basics preserved
- [ ] `npm run build` passes
- [ ] Changelog of key visual/UX changes captured

---

## Phase 6 — Visual Proof Loop (Every Round)
**Owner:** AX

### Checklist
- [ ] Desktop screenshot captured
- [ ] Mobile screenshot captured
- [ ] Side-by-side comparison vs previous round
- [ ] Sent to stakeholder for rapid review

---

## Phase 7 — Three Improvement Rounds
**Team:** Vanguard, Muse, Lens, Pulse, Quill, Atlas, Forge

### Round goals
- **R1:** Remove major quality issues
- **R2:** Raise differentiation + polish
- **R3:** Final premium finish and consistency

### Checklist per round
- [ ] Critic findings (P0/P1/P2)
- [ ] Design + visual updates
- [ ] Copy + interaction updates
- [ ] SEO/IA alignment updates
- [ ] Build passes
- [ ] Screenshots delivered

---

## Phase 8 — Final QA (Single Sentinel Pass)
**Owner:** Sentinel (runtime-capped)

### Checklist
- [ ] SEO basics pass
- [ ] Accessibility basics pass
- [ ] Link integrity pass
- [ ] Conversion clarity pass
- [ ] Only blocking issues + top 3 non-blocking improvements reported

---

## Overnight / Low-Input Mode
When stakeholder is offline:
- Team may continue up to the next gate using highest-scoring direction.
- If a decision is required, Vanguard scorecard decides tie-break.
- Keep a clear morning handoff:
  - What changed
  - Why it changed
  - Before/after screenshots
  - Open decisions
