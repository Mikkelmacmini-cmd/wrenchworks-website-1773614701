# Full Bay Media — Visual Refresh Concept Sprint

Date: 2026-03-11  
Owner: Muse (Visual)

## Goal
Create 3 distinct, brand-aligned visual directions using **white + green + red** for:
- Hero
- Service cards
- Proof rail
- CTAs

---

## Shared Brand Guardrails (applies to all 3 directions)

### Core palette anchors
- **White / Base:** `#FFFFFF`
- **Green / Trust + Positive action:** `#127A43`
- **Red / Urgency + primary conversion:** `#C62828`

### Supporting neutrals
- Charcoal text: `#1F2937`
- Soft gray surface: `#F5F7F6`
- Divider gray: `#D9E1DE`

### Color usage rule
- Red is for high-intent primary CTA and urgency labels only.
- Green is for trust, outcomes, proof highlights, and secondary interactive emphasis.
- White remains dominant background to preserve clarity and mobile readability.

## Mood Board 1 — **Precision Clean**

### Visual thesis
Clinical confidence: clean white canvas, sharp green cues, red reserved for “Book Now” moments.

### Image direction
- Bright daylight service bay shots (clean, organized, modern tools)
- Tight detail photos: diagnostics screen, calibrated tools, technician hands
- Customer handoff moments (keys, explained estimate, approval screen)
- Minimal clutter, high contrast, true-to-life color grading

### Palette stack
- `#FFFFFF` (80%)
- `#127A43` (12%)
- `#C62828` (6%)
- `#1F2937` (2%)

### Type scale
- Display/Hero: **48/56**, weight 700
- H2: **34/42**, weight 700
- Card title: **22/30**, weight 650
- Body: **17/27**, weight 400
- Meta labels/chips: **13/18**, weight 600

Suggested pairing: **Manrope** (headlines) + **Inter** (body/UI)

### Sample layout pattern
- **Hero:** left copy + right photographic panel with soft green diagnostic grid overlay
- **Service cards:** white cards, green icon chips, red “Book” micro-CTA text link on hover
- **Proof rail:** horizontal cards with green “Verified” badge + timestamp

## AI vs Photo-Shoot Recommendation

### Recommendation: **Hybrid (best outcome)**

Use a **photo-shoot for hero + proof-critical trust imagery**, and AI generation for supporting/illustrative assets.

#### Photo-shoot required (high trust impact)
- Hero people images
- Team portraits
- Customer handoff moments
- Shop-specific environment shots

Why: authenticity directly affects conversion and credibility, especially for local service businesses.

#### AI-generated acceptable (cost/speed leverage)
- Iconography
- Abstract proof/metric illustrations
- Background textures/overlays
- Non-literal support visuals for service categories

Why: faster iteration and easier style consistency without risking authenticity where it matters most.

### Suggested split
- **60% photo-shoot assets** (conversion-critical)
- **40% AI-generated assets** (supporting visuals/system assets)

## Final Direction Pick Guidance

If the client wants:
- **Most premium + clean trust:** choose **Precision Clean**
- **Most human + local warmth:** choose **Human Workshop**
- **Most bold + modern performance vibe:** choose **Performance Signal**

**My recommended default:** **Precision Clean** as base, then borrow human photo moments from **Human Workshop** for better emotional trust.
