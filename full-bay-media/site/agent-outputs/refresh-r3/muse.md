# Refresh Round 3 (Muse) — Final Brand-Lock Design Pass

## Objective
Deliver a homepage and service-system direction that is unmistakably **premium, bold, trustworthy**, and no longer template-coded. This pass integrates:
- Round 1 foundations (clarity + premium tone)
- Round 2 hierarchy/spacing upgrades
- Vanguard’s final critique (proof depth, mobile conversion mechanics, decision-engine behavior)

This is the final visual + layout guidance for implementation.

---

## 1) Final Creative Thesis (What makes this ownable)
AutoTrek should feel like a **precision diagnostic operator**, not a generic local shop site.

### Signature visual language (required)
Adopt one ownable motif across hero, cards, process, and CTA surfaces:

**“Diagnostic Pulse Grid”**
- A subtle technical grid baseline (low-contrast, dark-on-dark) that anchors major sections.
- Thin amber pulse line used to indicate decision progress (diagnose → approve → repair → verify).
- Appears as a structural accent, not decoration.

If this motif is inconsistently applied, the experience regresses into template territory.

---

## 2) Final Hierarchy System (Conversion priority lock)

## Priority order (must match visual weight)
1. **Book Inspection** (primary conversion)
2. Outcome promise (clear plan before costly failure)
3. Verifiable trust proof (rating, years, warranty, credentials, real outcomes)
4. Symptom triage quickstart (decision support)
5. Process/risk-control proof (approval gate)
6. Deep validation (reviews, team, comparison, FAQ)

## Enforcement rules
- One and only one filled CTA style per viewport.
- Secondary actions must be text/outline and visibly subordinate.
- Trust proof must be specific (who, when, what outcome), not abstract claims.
- Every section must answer one user anxiety. If not, cut it.

## 4) Spacing Precision (Tokenized, premium rhythm)
Base unit: 8px.

## Section rhythm
- Hero: `pt-28 pb-24`
- Core sections: `py-24 md:py-28`
- Section transitions: `mt-12 md:mt-16`
- High-stakes conversion zones (evidence, triage, final CTA): add +8–16px pre-space

## Internal rhythm
- Section stack: `gap-8 md:gap-10`
- Card/grid gutters: `gap-6 md:gap-8`
- Card padding: `p-6 md:p-8`
- CTA cluster: `gap-4`
- Label/meta clusters: `gap-2`

No ad-hoc spacing values. All spacing maps to token scale.

## 5) Trust Architecture by Anxiety Stage (Final)

## Stage A: “Can I trust this shop at all?” (Hero + immediate evidence)
Place under hero CTA:
- 4.9★ Google (with review count)
- Serving Littleton since 2009
- ASE-certified technicians
- 3yr/36k warranty

Add one real snippet immediately below chips:
- “2018 Audi Q5 coolant leak diagnosed same day, approved estimate, repaired next morning.”

## Stage B: “Will I get upsold?” (Process guardrail)
Dominant statement:
**“No repairs begin until you approve the estimate.”**

Support with micro-proof:
- timestamped estimate workflow
- approval checkpoint before wrench time

## Stage C: “Will they solve my exact issue?” (Triage + service detail)
- Symptom-led quickstart outputs recommended action
- Driveability guidance (safe now / caution / stop driving)
- Typical time window + budget band (ranges, not promises)

## Stage D: “Are people like me getting good outcomes?” (Validation)
- 3–5 short verified review excerpts with vehicle + issue + result
- Recent repair outcomes strip with turnaround time

## Stage E: “What do I do right now?” (Persistent action)
- Sticky mobile action rail: `Call Now`, `Request Callback`, `Get Quick Diagnosis`
- Business-hours state + after-hours callback capture

## 6) Component Specs (Implementation-ready)

## Header
- Clean nav: Services, Reviews, About, Contact
- Right cluster: phone + `Book Inspection`
- Height: 72 desktop / 64 mobile
- Backdrop: dark translucent with contrast-safe blur

## Hero
- H1 max 12–14 words, max 2 lines desktop
- Subcopy max 2 lines desktop
- CTA row: primary + secondary only
- Trust chips directly below CTA row
- Right media must be authentic shop imagery (no stock feel)

## Mobile sticky action rail (P0)
- Persistent on mobile across home + service templates
- Three actions:
  1) Call Now
  2) Request Callback
  3) Quick Diagnosis
- 56–64px touch targets, clear active/focus states

## Symptom triage quickstart (P0)
2–3 steps max:
1) Symptom selection
2) Urgency check
3) Recommended next action + prefilled booking/call path

## Evidence strip (replaces generic trust row)
- Real review snippet cards (short, scannable)
- Credential chips with plain-language meaning
- Warranty one-liner + exclusions link

## Service decision blocks (P0 on service pages)
Required blocks per key service:
- Common Symptoms
- Typical Causes
- Driveability Guidance
- Typical Time Window
- Budget Range (banded)
- What Can Wait vs What Cannot

## Comparison module
“AutoTrek vs Dealer vs Chain”
- factual comparison only
- dimensions: diagnostic depth, approval transparency, communication speed, warranty posture

## Team identity layer
- Advisor/technician cards (photo, credential, years, specialty)
- Keep concise; objective is trust realism, not biography length

## 7) Typography + color governance (single source lock)

## Type
- H1: condensed/authoritative, slightly tightened tracking
- H2: high-contrast utility style
- Body: `text-base md:text-lg`, max `68ch`
- Meta: never below accessible contrast threshold

## Color roles
- Foundation: `#0B0F14`, `#121821`
- Ink: `#EAF0F6`
- Muted: `#A9B4C2`
- Accent: `#C89B5E` (primary CTA, pulse line, key numerals only)

No extra accent families. No bright promotional color drift.
