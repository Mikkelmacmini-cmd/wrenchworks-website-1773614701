# AutoTrek — Overnight Concept Sprint (Muse)

Aligned to SOP Phase 2 (3 distinct directions), with brand lock: **Premium / Bold / Trustworthy**.

---

## Concept A — Precision Command
**Positioning feel:** High-performance diagnostic studio. Crisp, technical, decisive.

### 1) Hero composition
- **Layout:** 2-column asymmetric split (58/42 desktop, stacked mobile).
- **Left (decision column):**
  - H1 with diagnosis-first promise
  - One-sentence proof-led subhead
  - CTA row: `Call Now` (primary) + `Request Appointment` (secondary)
  - Trust chips directly under CTA: `4.9★`, `600+ Reviews`, `Since 2009`, `ASE-Certified`, `Warranty-Backed`
- **Right (evidence panel):**
  - Real shop photo with dark gradient overlay
  - Floating “Live Proof” cards (recent repair outcomes, avg turnaround, approval-first note)

### 2) Typography system
- **Display:** Sora or Space Grotesk, 700–800, tight tracking for bold authority
- **Body/UI:** Inter, 400–600 for high readability
- **Scale:**
  - H1: 56/60 desktop, 38/44 mobile
  - H2: 40/46 desktop, 30/36 mobile
  - Body: 18/30 desktop, 17/28 mobile

### 3) Spacing rhythm
- 8px system with premium air:
  - Section vertical: 112 desktop / 72 mobile
  - Card padding: 28–32 desktop / 20–24 mobile
  - CTA cluster spacing: 16–20
- Rule: add extra pre-space before conversion pivots (proof strip, process, final CTA)

### 4) Color direction
- **Base:** Deep graphite/navy (#0F172A, #111827)
- **Trust:** Slate steel (#334155, #475569)
- **Action accent:** Electric cyan-blue (#0EA5E9)
- **Success cue:** Controlled emerald (#10B981)
- High contrast, low-saturation surfaces; accent only on actions and key numerals.

### 5) Section architecture
1. Hero (promise + immediate proof)
2. Evidence strip (metrics + credentials)
3. “How We Work” diagnostic timeline (Diagnose → Confirm → Repair → Verify)
4. Service decision grid (symptom-led entry)
5. AutoTrek vs Dealer vs Chain (comparison table)
6. Recent outcomes (specific, verifiable repairs)
7. Final CTA with risk-reversal language

### 6) Component language
- **Cards:** squared corners with subtle radius (12–14), sharp edges feel engineered
- **Buttons:** high-contrast filled primary, outline secondary, strong focus ring
- **Chips:** smoked-glass capsules for proof facts
- **Tables:** clear row separators, sticky first column on mobile-horizontal scroll
- **Icons:** monoline technical icons, low decorative weight

## Concept B — Human Expert Ledger
**Positioning feel:** Boutique trusted advisor. Calm, premium, accountability-first.

### 1) Hero composition
- **Layout:** full-width cinematic image with anchored content block (left bottom on desktop, top on mobile).
- **Hero narrative:** technician + customer consultation moment (human trust, not anonymous machinery).
- **Content block:**
  - Headline around clarity + no-surprise approvals
  - Subhead emphasizing transparent decisions
  - Primary CTA and “Talk to an Advisor” micro-CTA
  - Credential line with headshots/initials of lead advisors

### 2) Typography system
- **Display:** Manrope or General Sans, 700 (warmer but still modern)
- **Body:** Inter, 400/500
- **Tone:** slightly more open tracking and generous line-height for calm confidence
- **Scale:**
  - H1: 52/60 desktop, 36/42 mobile
  - H2: 36/44 desktop, 28/34 mobile
  - Body: 18/30

### 3) Spacing rhythm
- Larger negative space, editorial pacing:
  - Section vertical: 120 desktop / 80 mobile
  - Max content width: 1200 with narrower text measure (62–68ch)
  - Intentional breathing room around testimonials/proof blocks

### 4) Color direction
- **Base:** Midnight blue (#0B1220)
- **Surface:** Warm slate (#1E293B)
- **Premium neutral:** Soft stone (#E5E7EB / #CBD5E1)
- **Accent:** Amber-gold restraint (#F59E0B) for badges/critical proof
- **CTA:** deep blue with white text (no neon)

### 5) Section architecture
1. Hero (advisor-led reassurance)
2. Trust ledger row (credentials, warranty terms, review volume)
3. “Meet Your Service Team” mini-profiles
4. Process + approval checkpoints (what you approve, when)
5. Service categories with “When to choose this” guidance
6. Story-backed reviews (vehicle, issue, fix, outcome)
7. Closing reassurance module + contact actions

### 6) Component language
- **Surfaces:** soft-elevated cards, subtle border glow, mild blur overlays
- **Proof modules:** ledger style (label/value with timestamps)
- **Testimonials:** quote + repair receipt metadata style
- **CTA bar:** persistent mobile sticky with call + appointment split
- **Imagery:** people-first moments, then technical support visuals

## Concept C — Bold Motion Garage
**Positioning feel:** Premium modern shop with assertive energy. Distinctive and memorable.

### 1) Hero composition
- **Layout:** layered diagonal composition with motion lines (static-safe if reduced motion).
- **Top line:** urgency utility chip (“Same-day diagnostics on most common issues”) 
- **Core block:** punchier bold headline + direct CTA stack
- **Right rail:** animated (or static fallback) status tiles:
  - “Now booking this week”
  - “Approval before repair”
  - “Warranty-backed work”
- Mobile collapses to a strong vertical stack with persistent sticky CTA.

### 2) Typography system
- **Display:** Archivo Expanded / Eurostile-like modern alternative for high-impact headers
- **Body/UI:** Inter
- **Scale:**
  - H1: 60/62 desktop, 40/44 mobile
  - H2: 42/48 desktop, 32/38 mobile
  - Body: 17/28
- Strong weight contrast: headlines heavy, metadata compact and precise.

### 3) Spacing rhythm
- Tighter than A/B but controlled:
  - Section vertical: 96 desktop / 64 mobile
  - Intra-module rhythm: 12/16/24 cadence
  - Aggressive fold optimization: most decision-critical proof appears within first 1.5 scrolls

### 4) Color direction
- **Base:** Carbon black (#0A0A0A)
- **Secondary:** Gunmetal (#1F2937)
- **Accent 1:** Signal red-orange (#F97316) for urgency/action
- **Accent 2:** Ice blue (#38BDF8) for trust/technical labels
- Must enforce strict contrast to avoid “sporty but unreadable” failure.

### 5) Section architecture
1. Hero (high-energy value + immediate action)
2. Quick triage selector (“What are you experiencing?”)
3. Trust proof matrix (ratings, certifications, warranty, years)
4. Service performance cards (problem → fix pathway)
5. Comparison module (dealer/chain/AutoTrek)
6. FAQ as decision unblocker (cost timing approvals)
7. High-contrast final conversion band

### 6) Component language
- **Buttons:** chunky, high-contrast, subtle press-state depth
- **Cards:** angular cut-corner motif for brand distinctiveness
- **Status tiles:** compact, live-dashboard visual metaphor
- **Badges:** stronger visual weight but limited to proof/action only
- **Micro-motion:** 120–220ms transitions, no decorative looping near CTAs

---

## Cross-Concept Mobile-First Rules (applies to A/B/C)
- Sticky bottom conversion bar on mobile: `Call` + `Book`.
- Hero must communicate **promise + action + trust proof** in first viewport.
- Trust data remains text-rendered (not image-only) for accessibility/SEO.
- Touch targets minimum 44px; primary CTA always visually dominant.
- Reduced-motion mode preserves hierarchy with static states.

## Distinctiveness Check (SOP requirement)
- **A (Precision Command):** technical, systemized, engineered confidence.
- **B (Human Expert Ledger):** relationship-led, advisor trust, premium calm.
- **C (Bold Motion Garage):** assertive, energetic, brand-memorable conversion design.

These are intentionally not palette swaps — each direction changes composition logic, pacing, and trust expression model.
