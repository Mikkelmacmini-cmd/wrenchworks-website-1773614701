# AutoTrek Refresh — Round 2/3 Critic Pass (Vanguard)

Date: 2026-03-11  
Mode: Stricter gate, market-dominance standard (not “better than average”)

## Executive Verdict
Round 1 improved aesthetics, but this implementation is still a **premium skin over a conventional local-service template**.  
It does **not yet feel unmistakably 2026**.

Current state: **credible and cleaner**.  
Required state: **category-defining, conversion-engineered, proof-dense, and ownable**.

If shipped as-is, this likely lifts baseline trust but leaves major conversion and differentiation upside on the table.

---

## Updated Scorecard (strict)
- **Premium feel:** 7.3/10 (good palette/type, still component-generic)
- **Boldness:** 6.8/10 (headline is strong, rest of page rhythm is safe)
- **Trustworthiness:** 6.4/10 (claims exist, hard proof depth is thin)
- **2026 distinctiveness:** 5.1/10 (looks modern, not ownable)
- **Mobile conversion mechanics:** 5.7/10 (missing persistent intent capture)
- **Decision acceleration:** 5.9/10 (insufficient tools for “what should I do now?”)

**Overall:** **6.2/10** — respectable, still not dominant.

---

## What Improved Since R1
1. Visual coherence is materially better (dark/warm system, cleaner hierarchy).
2. Hero clarity and CTA language are stronger.
3. Process framing (Diagnose → Confirm → Repair → Verify) remains a strong trust backbone.
4. Service detail pages are cleaner and easier to scan.

These are real gains. They just don’t cross the “unmistakably premium 2026” threshold.

---

## Core Gaps (Why this still feels template-like)
1. **Proof is asserted, not evidenced.**  
   “4.9★” and warranty appear, but no verified review excerpts, timestamping, credential artifacts, advisor identity, or before/after repair outcomes.

2. **No high-intent micro-funnels.**  
   Users can call, but cannot quickly triage symptoms, submit context, get urgency guidance, or choose appointment type in <30 seconds.

3. **Homepage section rhythm is predictable.**  
   Hero → trust boxes → cards → process → two cards → CTA. Good structure, low memorability.

4. **Service pages lack hard decision tools.**  
   “Timeline/pricing posture” is generic. Missing typical ranges, urgency matrix, “safe to drive?” logic, and common-failure signatures.

5. **Brand governance drift risk.**  
   Design direction docs diverge (e.g., type guidance differences, accent philosophy variance). Without one locked token source, future edits will degrade consistency.

6. **Mobile-first conversion not fully engineered.**  
   No sticky bottom action rail on key templates; no one-thumb escalation path for urgent users.

---

## P0 — Must Ship Before Calling This “2026 Premium”
1. **Install a persistent mobile action rail (home + services + detail).**  
   Actions: `Call Now`, `Request Callback`, `Get Quick Diagnosis`.  
   Include business-hours state + “after-hours callback” capture.

2. **Replace claim-only trust row with evidence modules.**  
   - 3–5 real review snippets (vehicle + problem + outcome).  
   - Certifications with plain-language meaning.  
   - Warranty explained in one sentence with exclusions link.

3. **Ship Symptom Triage Quickstart (interactive, 2–3 steps).**  
   - What are you noticing?  
   - Is it urgent/safety-critical?  
   - Recommended next action + prefilled booking/call path.

4. **Upgrade service pages with decision-critical data blocks.**  
   Required blocks: `Common Symptoms`, `Typical Causes`, `Driveability Guidance`, `Typical Time Window`, `Budget Range (banded)`, `What can wait vs what cannot`.

5. **Add “AutoTrek vs Dealer vs Chain” comparison.**  
   Keep factual and calm. Show communication speed, approval transparency, warranty posture, diagnostic depth.

---

## P1 — High-Leverage Differentiation (next sprint)
1. **Create one ownable visual motif and apply consistently.**  
   Example: precision-grid + diagnostic pulse line language that appears in hero, cards, process, and CTA surfaces.

2. **Introduce real operator identity layer.**  
   Advisor/technician cards with credentials, years, specialties, and human photo proofs.

3. **Add “Recent Repair Outcomes” strip.**  
   Mini case cards: vehicle, issue, fix, turnaround, customer result.

4. **Contextualize CTAs by user intent.**  
   Replace repetitive “Book Inspection” with intent-specific actions (`Check Engine Diagnosis`, `Brake Safety Check`, `Hybrid Battery Evaluation`).

5. **Tokenize brand system in a single source of truth.**  
   Lock typography, color usage ratios, component spacing, and interaction rules in one implementation-facing doc + CSS token map.

---

## P2 — Polish + Moat
1. **Micro-interaction refinement pass.**  
   Add deliberate motion hierarchy (hover, reveal, CTA transitions) without gimmicks.

2. **Localized authority proof.**  
   Most-serviced makes/models in Littleton, service radius map, pickup/dropoff policy clarity.

3. **Media authenticity upgrade.**  
   Replace generic/staged visuals with real shop imagery/video snippets (inspection screen, bay workflow, handoff moment).

4. **Conversion experimentation framework.**  
   A/B test offer framing and CTA order by page intent.

---

## Non-Negotiable Gate for Round 3
Do not call final unless all are true:
- Proof is **specific and verifiable**, not just claims.
- Mobile user can take decisive action in one thumb-zone at all times.
- Service pages help users decide **now**, not just “learn more.”
- Visual language contains at least one **ownable signature** beyond palette.
- Experience feels like a high-confidence operator, not a polished template.

---

## Bottom Line
This is a strong intermediate rebuild, not the end state.  
To feel unmistakably **2026 + premium/bold/trustworthy**, AutoTrek must move from **presentation quality** to **decision-engine quality**: evidence, triage, intent routing, and ownable brand mechanics.
