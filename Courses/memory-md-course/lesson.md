Lesson: What is MEMORY.md?

1) Simple definition
MEMORY.md is a place to write down "how we do things" — not facts about the world, but the habits, preferences, patterns, and lessons a person or team uses. It's like a cookbook for how you think and work.

2) Why it matters
- Stops us from repeating mistakes
- Helps new people learn how we like to work
- Keeps small but important knowledge from disappearing

3) What goes in MEMORY.md
- Patterns: "We always run tests before merging"
- Preferences: "Use 2-space indentation" or "Morning standups at 9:15"
- Lessons learned: "Deploying on Fridays caused outages; avoid it"
- Decision rules: "If two team members disagree, the owner decides"

4) What does NOT belong
- Random facts about the world (use items.json or notes)
- Sensitive secrets (passwords, tokens)

5) Easy example (story)
Imagine you and a friend build a treehouse. After some mistakes, you write down: "Measure twice, cut once" and "Always wear gloves when using the saw." That's MEMORY.md — lessons so you don't forget next time.

6) How to read and use it
- Read MEMORY.md when onboarding someone
- Check it before making big changes
- Update it when you learn something new about how you work

7) Short checklist for updates
- Is this about how we work or a world fact? If how we work → add it
- Is it simply replacing an older preference? Mark the old one superseded
- Is it time-sensitive? Note the date and reason
